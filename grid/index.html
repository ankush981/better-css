Size values

When we first learn CSS Grid, we come across code such as grid-template-columns: repeat(4, 1fr); and we resign ourselves to mystery. But repeat and fractional sizes are not our only recourse.

CSS Grid is not just very powerful but also very friendly. When generating rows, columns, etc., we can use the units and precise values we want, completely ignoring the weird fr thing.

Allow me to lift some code shamelessly from the CSS Tricks folks: 

                    .container {
                           grid-template-columns: 40px 50px auto 50px 40px;
                           grid-template-rows: 25% 100px auto;
                    }

This can be visualized as (again, thanks to CSS Tricks):

As you can see, the units specified for row and column widths here are our regular, everyday units: px, %, auto, etc. The fr stuff is not necessarily needed.

fr strikes back!

From the earlier section, one might conclude that fr is just a minor convenience or syntax sugar.

Far from it!

I'll spare you the code, so let's understand the idea conceptually. When we want to define the grid widths manually, there's a chance we might screw up. For instance, something innocent as grid-template-columns: repeat(4, 25%); can produce horizontal scrolling if there are other elements.

However, if we say grid-template-columns: repeat(4, 1fr);, we're leaving it up to CSS Grid to take everything into consideration and just gives us four equal-width columns.

So, yes, in the vast majority of cases, fr; is the correct unit.
